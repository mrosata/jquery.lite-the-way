Lite the Way

Lite the Way, a free to abuse jQuery plugin created with the intention to make reading long content easier on the eyes. Lite the Way works best on large text containers of text and images by toggling active and faded css classes on any targeted elements. It determines which elements are active or faded based on whether they fall in the middle(active) or above/below(faded) invisible margin lines created by Lite the Way. You can define and tweak your own boundaries, Lite the Way is highly customizable and as easy to use as calling out $('p').litetheway(). Get comfortable and customize as many options yourself by simply passing in a configuration object. Go ahead! Give it a try and then press the button!. Please note: the button hasn't been made active yet, it's not part of the plugin, when implemented, it will just be a fun easy way for you to experiement with plugin settings. So start reading or start scrolling!

 Configuration options    

Option
Default
Description
Option: context
Default: document
The plugin container. Use a CSS selector or the plugin will run over the entire page. By giving the plugin a context you can tell Lite the Way where it is allowed to toggle active and faded classes and display overlays if enabled. This demo page uses class name '.demo-text' for the context, which is acceptable, however, context would work best as an ID. They are the safest option as it limits the plugin to one area.
Option: start
Default: 140
The amount of pixels after the top edge context container that user must scroll before Lite the Way will apply activeClass and fadedClass names to elements the plugin was called on.
Option: end
Default: 140
How many pixel from the bottom edge of context area should lite the way be enabled. This is one of the few values at the moment that is measured by the bottom edge of the screen
Option: auto
Default: 0.25
( unless false, autoMargin will recalculate the margin for active text based on screenheight / auto if greater then 1, else screenheight * auto. For example: 0.25 would be 25% of the screen height, so would 5 because dividing by 4 and multiplying by .25 are both legitimate ways to calculate percentages. If auto is set to false then a fixed width will be used
Option: margin
Default: 290
if autoMargin is false, this will be the fixed width margin size used in px. 290 covers about 25% a normal browser window height. Because it's a measurement from the center up and down, realistically it's a 270px margin
Option: active
Default: 'active-class'
The name of the css class apllied on elements that fall within the margin
Option: faded
Default: 'faded-class'
The name of the css class applied to elements outside of the center margin
Option: overlays
Default: divue
Whether or not to create div overlays ( at current time, overlays make text unselectable when active )
Option: overlayClass
Default: 'lite-the-way-overlay'
The classname to use on the overlay divs (If you want overlay divs)
Option: overlayPadding
Default: 10
If overlays are visible, this will extend their borders, in case you don't want the overlays to start and stop exactly where the active and faded text meet.
Option: topMult
Default: 1
Move the top line of the margin up or down by a percentage using multipliers. 1.5 would be 150% the size of regular top margin
Option: bottomMult
Default: 1
Same as topMult but for the bottom, another example, 0 would be 0%, so the bottom margin line would end at exactly center screen
Option: cutoff
Default: false
An element is normally active if it's top edge falls within main margins. Set cutoff to divue, and lite the way will also measure by the bottom edge as well. However, note: at this time, that means any element with a top edge and bottom edge outside of the margins will be deactive. So paragraphs larger then the active area may never display active.

This was instantiated using the following code:
// Set the plugin on paragraphs, only within container '.demo-text'
$('*').litetheway({
    document : '.demo-text',
    topMult : 1.25, // top margin line is increased by 25% using
    cutoff : true, // If elements bottom edge falls below margin, don't display it as active
    start : 240, // Start the plugin when top of window gets 160px down
    end : 80 // Stop the plugin when bottom of window gets 260px down
});

By clicking this link, a test done on start and end margins (area is shaded) will activate a darkish blue area at top and bottom of the demo area .demo-text container. Lite the way, a plugin that allows you to easily create lighted up reading areas for your sites vistors in areas of large portions of text. The plugin is intended to make reading easier on the users eyes and not strain their vision. Using a set of margins which can be created using fixed width or screen percentage, paragraphs will lighten(ativate) or darken(deactivate) depending on whether or not they fall outside or inside of the margins.

You are able to style your own active and inactive classes as well as overlays if you decide to use the overlay effect (part of the plugin as well). To get an idea for how Lite the Way works, use whichever means of scrolling you prefer and navigate this webpage up or down. Whenever you are more then the set start distance from the top of this container, the paragraphs that fall outside of the defined margins will recieve classname 'fade-out-read' by default.

So this plugin will be usable by simply declaring $('p').liteTheWay( options ) where 'p' could actually be any type of selector you want to be effected by the plugin, and options will be a JavaScrip object that allows you to override default settings such as top and bottom starting points, top and bottom margins for when 'Lite the Way' is active, custom classnames for elements that fall within the margins and for ones that fall outside the margins, margin top or bottom multipliers, colors for the overlays and even callback functions for certain plugin events are all possible to set, but completely optional. Lite the Way works very well with minimal settings too.

Some areas that need fixing... If Paragraphs become to large, then the chance that a paragraph could start and stop outside the margins set by plugin increase. This would mean only faded text. I am thinking of a solution for this. Most likely, a seperate option that will be on by default, and it will check for either this condition, or check that no paragraphs have the active class. Either way, there could be a dynamically injected <span> element that can wrap a portion of a paragraph with the active class. This can also solve an issue where over half the text in active margin is faded.
