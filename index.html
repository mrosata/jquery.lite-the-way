Lite the Way

This is a jQuery plugin created to light up the general area of reading within a page. It is highly customizable and as easy as calling out $('p').litetheway() or you can customize many options yourself. Go ahead! Give it a try and then press the button!. Please note: the button hasn't been made active yet, it's not part of the plugin, when implemented, it will just be a fun easy way for you to experiement with plugin settings. So start reading or start scrolling!

  • autoMargin : default = 0.25, ( unless false, autoMargin will recalculate the margin for active text based on screenheight / autoMargin if greater then 1, else screenheight * autoMargin. For example: 0.25 would be 25% of the screen height, so would 5 because dividing by 4 and multiplying by .25 are both legitimate ways to calculate percentages.
  • fixedMargin : default = false, if autoMargin is false, this is the set margin on all screen sizes. ( Not setup yet, sorry )
  • margin : default = 135, if autoMargin is false, this will be margin size used in px. 135 covers about 20% a normal browser window height. Because it's a measurement from the center up and down, realistically it's a 270px margin
  • start : default = 240, How many pixels after sthe top of the context area should lite-the-way be enabled and begin to add faded and active classes to elements. Unless otherwise stated, this number also defines 'end'
  • end : default = 240, How many pixel from the bottom edge of context area should lite-the-way be enabled. This is one of the few values at the moment that is measured by the bottom edge of the screen
  • active : default = 'active-text', The name of the css class apllied on elements that fall within the margin
  • faded : default = 'fade-out-text', The name of the css class applied to elements outside of the center margin
  • topMult: default = 1, Move the top line of the margin up or down by a percentage using multipliers. 1.5 would be 150% the size of regular top margin
  • bottomMult : default = 1 Same as topMult but for the bottom, another example, 0 would be 0%, so the bottom margin line would end at exactly center screen
  • cutoff : default = false, An element is normally active if it's top edge falls within main margins. Set cutoff to true, and lite-the-way will also measure by the bottom edge as well. However, note: at this time, that means any element with a top edge and bottom edge outside of the margins will be deactive. So paragraphs larger then the active area may never display active.
  • overlays : default = true, Whether or not to create div overlays ( at current time, overlays make text unselectable when active )
  • overlayPadding : default = 10, If overlays are visible, this will extend their borders, in case you don't want the overlays to start and stop exactly where the active and faded text meet.
  • overlayClass : default = 'litetheway-overlay'
  • document : default = document, which element to begin offset from ( so far only tested with the default, no idea how it works otherwise )

Config using »

This was instantiated using the following code:
// Set the plugin on paragraphs, only within container '.demo-text'
$('p').litetheway({
    document : '.demo-text',
    topMult : 1.25, // top margin line is increased by 25% using
    start : 160, // Start the plugin when top of window gets 160px down
    cutoff : true, // If elements bottom edge falls below margin, don't display it as active
    end : 260 // Stop the plugin when bottom of window gets 260px down
}); ( The darkish blue area at top and bottom of the .demo-text area is not part of the plugin, it is to demonstrate how accurate the plugins activation and deactivation margin is. The top one is set at 160px from top of .demo-text and the bottom line at 260px. )

What is this?

Lite the way, a plugin that allows you to easily create lighted up reading areas for your sites vistors in areas of large portions of text. The plugin is intended to make reading easier on the users eyes and not strain their vision. Using a set of margins which can be created using fixed width or screen percentage, paragraphs will lighten(ativate) or darken(deactivate) depending on whether or not they fall outside or inside of the margins. You are able to style your own active and inactive classes as well as overlays if you decide to use the overlay effect (part of the plugin as well). To get an idea for how Lite-the-Way works, use whichever means of scrolling you prefer and navigate this webpage up or down. Whenever you are more then the set 'start' distance from the top of this container, the paragraphs that fall outside of the defined margins will recieve classname 'fade-out-read' by default. So this plugin will be usable by simply declaring $('p').liteTheWay( options ) where 'p' could actually be any type of selector you want to be effected by the plugin, and options will be a JavaScrip object that allows you to override default settings such as top and bottom starting points, top and bottom margins for when 'Lite the Way' is active, custom classnames for elements that fall within the margins and for ones that fall outside the margins, margin top or bottom multipliers, colors for the overlays and even callback functions for certain plugin events are all possible to set, but completely optional. Lite-the-Way works very well with minimal settings too.

Some areas that need fixing... If Paragraphs become to large, then the chance that a paragraph could start and stop outside the margins set by plugin increase. This would mean only faded text. I am thinking of a solution for this. Most likely, a seperate option that will be on by default, and it will check for either this condition, or check that no paragraphs have the active class. Either way, there could be a dynamically injected <span> element that can wrap a portion of a paragraph with the active class. This can also solve an issue where over half the text in active margin is faded.

Why?
Why you ask?

This plugin must be able to know where it is needed on a page and where it is not needed on a page. If it didn't know the page real estate location of it's own context area, then it would end up forcing the user to define offsets from top and bottom of the page just to attempt to make the plugin only work on the area in which their text to be lite-the-way-ifyed is. This would suck, bad, because there is no telling how tall the context area might be on different sized displays. We don't want lite-the-way plugin users to design their sites to suite lite-the-way, we want lite-the-way users to out of the box give a great ux to their page by only having to type one line jQuery('p').litetheway({document:'#myTextHolder'});
To test that out, we have added this large portion of text to the bottom of this page to make sure that our code can figure out where the .coolbeans element starts and stops. Thanks for using, sorry for reading!
